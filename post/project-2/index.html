Projeto 2: Detecção de Cliques Fraudulentos

July 9, 2020

Foram analisados dados públicos da maior plataforma independente de serviço de big data da China que cobre mais de 70% dos dispositivos móveis ativos em todo o país. Eles processam 3 bilhões de cliques por dia, dos quais 90% são potencialmente fraudulentos. A partir do dados históricos foi criado um modelo de classificação que avalia se um usuário fará o download de um aplicativo depois de clicar em um anúncio de aplicativo para celular.

Link para o repositório Github

Introdução

A maior plataforma independente de serviço de big data da China, cobre mais de 70% dos dispositivos móveis ativos em todo o país. Eles processam 3 bilhões de cliques por dia, dos quais 90% são potencialmente fraudulentos. Os canais de anúncios podem aumentar os custos simplesmente clicando no anúncio em larga escala. Com mais de 1 bilhão de dispositivos móveis inteligentes em uso ativo todos os meses, a China é a maior mercado móvel no mundo e, portanto, sofre com grandes volumes de tráfego fradulento.

O problema de negócio:

O risco de fraude está em toda parte, mas para as empresas que anunciam on-line, a fraude de cliques pode ocorrer em um volume esmagador, resultando em dados de cliques enganosos e desperdício de dinheiro. A abordagem atual para evitar a fraude de cliques para desenvolvedores de aplicativos é medir a jornada do clique de um usuário e sinalizar endereços IP que produzem muitos cliques, mas nunca acabam instalando aplicativos.

Objetivo do Trabalho:

Criar um modelo de classificação que avaliará se um usuário fará o download de um aplicativo depois de clicar em um anúncio de aplicativo para celular.

Linguagem Utilizada:

Este trabalho foi realizado com a linguagem R. Para detalhes de código acessar o link do github.

Kaggle e Fonte de Dados

Os dados trabalhados nesse projeto são públicos. Eles foram retirados do site Kaggle. Este site é amplamente utilizado por cientistas de dados que querem colocar o seu conhecimento à prova. Foi utilizado o dataset train_sample, que possui menos registros, pois não estou utilizando nenhuma técnica de Big Data. Portanto, foi feita uma análise estatística inferencial.

Dicionário de Dados:

Variável Descrição

Desenvolvimento

Primeiramente foram carregados o arquivo train_sample.csv na memória do computar utilizando a ferramente RStudio.

Feature Selection (Seleção de Variáveis):

Após os dados serem carregados. A primeira etapa para solução foi aplicar o que chamamos de Seleção de Variáveis. Foram adicionadas as seguintes colunas no dataset:

  • wday = Dia da semana em que o anúncio foi clicado
  • hour = hora do clique
  • nip_day_h = Número de cliques de um mesmo IP, no mesmo dia e na mesma hora
  • nip_h_chan = Número de cliques de um mesmo IP, no mesma hora e do mesmo canal de anúncio
  • nip_h_osr = Número de cliques de um mesmo IP, na mesma hora e de um mesmo OS
  • nip_h_app = Número de cliques de um mesmo IP, na mesma hora e no mesmo APP
  • nip_h_dev = Número de cliques de um mesmo IP, na mesma hora e no mesmo dispositivo

Esse processo é muito importante, pois conseguimos gerar novas informações a partir das informações disponíveis. Isso aumenta o nosso entendimento sobre o problema e nos permite gerar um modelo de predição ainda mais preciso. Também foram removidas as variáveis IP e click_time (pois essas informações agora estão nas novas colunas) e attributed_time (data do download).

Download Quantidade

Com essa análise percebemos que o dataset está altamente desbalanceado. Existe muito mais registros sobre cliques que não geraram download do quê cliques que geraram. Se treinarmos o nosso algoritmos com os dados desse jeito ele aprenderá muito mais sobre cliques que não resultam em download. Dessa forma estaríamos criando um modelo tendencioso e com uma taxa de erro maior.

Após essa conclusão apliquei uma técnica através de uma função da linguagem R que balanceia os dados.

Machine Learning

Analisando relevância das variáveis para o modelo preditivo

Aqui utilizei o algoritmo randomforest como ferramenta para averiguar a importância das variáveis para o modelo preditivo e em seguida foi feito um plot.

Com essa informação escolhi as 7 variáveis de maior relevância. Foi adotado o método Gini para a escolha das variáveis. Variáveis: app + channel + hour + device + os + nip_h_dev + nip_day_h.

Foram utilizados dois algoritmos de aprendizado de máquina: RandomForest(com e sem parametrização) e Supported Vector Machine(kernel linear e radial).

Resultados:

Algoritmo Acurácia
RF c/ parametrização 0.9573
RF s/ parametrização 0.956
SVM linear 0.6973
SVM radial 0.732

O algoritmo RandomForest não paremetrizado se mostrou melhor para a classificação dos cliques.

Após a seleção do algoritmo foi aplicada uma função que refaz a o processo de divisão aleatória dos dados em treino e teste e de treinamento do algoritmo por 20 vezes. Aplicando essa técnica consiguimos avaliar a precisão do modelo para diferentes dados de treino e teste.

Criando curvas ROC

A curva ROC, é uma representação gráfica que ilustra o desempenho de um sistema classificador binário. Quanto mais próximo de 1 na parte de cima da curva, melhor.

Conclusão

O modelo alcançou uma média de acurácia de 95% permitindo a conclusão do projeto. As estapas de seleção de variáveis e de balanceamento do dataset foram de estrema importância para que o modelo preditivo tivesse uma boa taxa de acertos. Além disso, o algoritmo RandomForest se sobressaiu em relação ao SVM.
