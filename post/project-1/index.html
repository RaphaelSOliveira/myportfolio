By Raphael Serra de Oliveira

O Grupo Bimbo, atualmente, é uma das mais importantes empresas de panificação do mundo. Este projeto teve como objetivo criar um modelo que preveja a demanda real de produtos para evitar a perda e devolução dos mesmos. Os dados consistem no conjunto de transações de vendas durante 7 semanas no México. O modelo contruído alcançou uma média de acurácia de 99%. O algoritmo de regressão linear se mostrou muito eficaz para este conjunto de dados.

Os detalhes de código desenvolvido você encontra no meu Github.

Introdução

O Grupo Bimbo foi fundado no México em 1945. Atualmente é uma das mais importantes empresas de panificação do mundo pelo posicionamento da marca, volume de produção e gigante número de vendas. Este projeto teve como objetivo prever a demanda real de produtos para evitar a perda e devolução dos mesmos.

O problema de negócio

A falta de precisão no cáculo da demanda de produtos gera desperdícios para a empresa. Os produtos podem tanto faltar acarretando a perda de uma compra como podem estragar antes de serem vendidos. Como a empresa possui produtos com prazo de validade curtos, a margem aceitável para erro é pequena.

Objetivo do Trabalho

Criar modelo de regressão que seja capaz de estimar a demanda de forma precisa.

Linguagem Utilizada

Este trabalho foi realizado com a linguagem R.

Kaggle e Fonte de Dados

Os dados trabalhados nesse projeto são públicos, estes consistem no conjunto de transações de vendas durante 7 semanas no México. Eles foram retirados do site Kaggle. Este site é amplamente utilizado por cientistas de dados que querem colocar o seu conhecimento à prova. Foi utilizada a técnica de estatística inferencial. A partir do dataset train.csv foi feita uma amostragem pois o data set original tem 3.2 GB e não estou utilizando nenhuma técnica de Big Data.

# Criando função que remove os outliers
remove_outliers <- function(x, na.rm = TRUE, ...) {
  qnt <- quantile(x, probs=c(.25, .75), na.rm = na.rm, ...)
  H <- 1.5 * IQR(x, na.rm = na.rm)
  y <- x
  y[x < (qnt[1] - H)] <- NA
  y[x > (qnt[2] + H)] <- NA
  y
}

# Removendo os outliers
train_sample_outlier <- remove_outliers(train_sample$Demanda_uni_equil)

# Visualizando a variável sem outliers

Machine Learning

Dividindo o dataset em treino e interessante

smp_size <- floor(0.70 * nrow(train_sample))
train_ind <- sample(seq_len(nrow(train_sample)), size = smp_size)

train <- train_sample[train_ind,]
test <- train_sample[-train_ind,]

Primeiramenteos os dados que retiramos para amostra foram divididos em treino e teste.

Analisando relevância das variáveis para o modelo preditivo

Foi utilizado o algoritmo randomforest como ferramenta para averiguar a importância das vairiáveis para modelo preditivo e a partir disso foi feito o plot a seguir.

modelo_rf1 <- randomForest(Demanda_uni_equil ~ .,
                           data=train,
                           importance=TRUE)

varImpPlot(modelo_rf1,main="Importânica Variávies")

Foram selecionadas as 7 primeiras variáveis do método IncNodePurity. Variáveis: Venta_uni_hoy + Venta_hoy + Canal_ID + Producto_ID + Ruta_SAK + Cliente_ID + Agencia_ID.

Treinando modelo preditivo com o algorítimo de regressão linear

Após a seleção das variáveis foi treinado o modelo de regressão linear. A seguir uma tabela com o desempenho do modelo.

Conclusão

O modelo alcançou uma média de acurácia de 99% permitindo a conclusão do projeto. O algoritmo de regressão linear se mostrou muito eficaz para este conjunto de dados. Assim consguimos ajudar a empresa a alcançar o seu objetivo, ter menos gastos com disperdício e aumentar a sua lucratividade.
